# Website UI/UX and HCI Review Checklist

Use this file when a request needs a more structured review rubric or when multiple interface directions need to be compared consistently.

## Fast Triage

Ask these first:

1. What is the user trying to get done on this screen?
2. What is the one action the business most wants?
3. What could confuse, delay, or scare the user before that action?
4. What happens if the user makes a mistake or stops midway?
5. What changes on mobile?

## Heuristic Review Areas

### Clarity

- The page purpose is obvious within a few seconds.
- The main heading is specific, not generic.
- The primary call to action is visually and semantically dominant.
- Labels are plain-language and avoid internal jargon.

### Hierarchy

- The most important information appears first.
- Sections are chunked into digestible groups.
- Typography, spacing, and contrast create a clear reading path.
- Secondary actions do not compete with the primary action.

### Interaction

- Controls look interactive and behave consistently.
- Multi-step flows expose progress and the next step.
- Loading, success, empty, and error states are designed intentionally.
- Destructive actions have safeguards.

### Forms

- Only necessary inputs are requested.
- Field labels remain visible during completion.
- Validation is timely and specific.
- The user understands why sensitive data is required.

### Accessibility

- Headings, labels, and controls are semantically meaningful.
- Color is not the only carrier of meaning.
- Focus states are visible.
- Touch targets are comfortably tappable on mobile.
- Reading order and keyboard order are predictable.

### Trust

- Claims are supported by evidence, examples, or social proof.
- Pricing, commitments, and privacy implications are understandable.
- The interface avoids dark patterns and hidden costs.

## Recommended Critique Template

Use this format when reviewing a page or flow:

1. Goal
State the user task and likely success metric.

2. What works
Call out a few strengths worth preserving.

3. Main risks
List the highest-impact usability or conversion issues first.

4. Changes
Describe concrete modifications to layout, copy, interaction, or state handling.

5. Expected impact
Explain what user behavior should improve if the changes are implemented.

## Pattern Reminders

- Landing page: headline, proof, CTA, objection handling, low-friction next step.
- Dashboard: scanability, anomaly visibility, comparison, actionability.
- Checkout or signup: trust, progressive disclosure, error recovery, minimal friction.
- Content-heavy page: navigation anchors, summaries, skim cues, reading rhythm.
